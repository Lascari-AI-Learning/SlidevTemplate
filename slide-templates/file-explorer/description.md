# File Explorer Template

## Purpose
An interactive file explorer slide with a VS Code-inspired dark theme. Displays a tree sidebar with folder/file navigation and a syntax-highlighted code viewer. The presenter can click through files live during the presentation.

## When to Use
- Walking through a project's file structure
- Showing code from multiple files in context
- Demonstrating how a codebase is organized
- Any slide where you want to show real code files interactively

## Template Variables

| Variable | Required | Description | Example |
|----------|----------|-------------|---------|
| `{{dir}}` | Yes | Path to the file tree data folder (format: `<slide-folder>/filetree/<project-name>`) | `09-demo/filetree/my-app` |
| `{{title}}` | No | Optional title displayed above the explorer | `"Project Structure"` |
| `{{speaker_notes}}` | No | Speaker notes for the slide | `"Walk through each file..."` |

## Usage Example

```bash
npm run generate:slide -- --template=file-explorer --name=09-demo \
  --dir="09-demo/filetree/my-app"
```

With optional title:
```bash
npm run generate:slide -- --template=file-explorer --name=09-demo \
  --dir="09-demo/filetree/my-app" \
  --title="Project Structure"
```

## Data Setup

The file explorer reads real files from disk. After generating the slide, create your file tree data:

```
slides/<slide-name>/
├── index.md              (generated by template)
└── assets/
    └── filetree/
        └── <project-name>/   (your project files go here)
            ├── src/
            │   └── app.py
            ├── tests/
            │   └── test_app.py
            ├── requirements.txt
            └── README.md
```

Each file should contain real, representative code (10-30 lines recommended for readability).

## Visual Features
- Dark VS Code-inspired theme (always dark, regardless of slide theme)
- Syntax highlighting via Shiki (supports 20+ languages)
- File type icons from VS Code icon set
- Expandable/collapsible folder tree with smooth animations
- Fade transitions when switching between files
- Tab header showing current file with close button
- All folders expanded by default, first file auto-selected

## Supported File Types
Icons and syntax highlighting for: Python, TypeScript, JavaScript, Vue, HTML, CSS, SCSS, JSON, YAML, Markdown, Shell, Rust, Go, TOML, Docker, Makefile, and more.

## Notes
- The `dir` prop format is `<slide-folder>/filetree/<project-name>` (not a filesystem path)
- Files are loaded at build time via Vite's `import.meta.glob` — no runtime file access
- Keep individual files concise for presentation readability
- The explorer fills the full slide height; when a title is provided it takes the remaining space below
